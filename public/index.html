Framework and Database

  • Express with EJS Framework

    • Express Framework: Built the backend application using Express.js framework

  • MongoDB for User and Video Database

    • MongoDB for User and Video Database: Utilized MongoDB to store user information, videos, likes, and other related data.

Middleware and Authentication

  • User Middleware

    • Implemented middleware for user authentication to manage user sessions and access control.

  • JWT Authentication

    • Employed JSON Web Tokens (JWT) for user authentication, utilizing both access and refresh tokens.

  • Cookie Storage

    • stored access and refresh tokens in cookies for enhanced security and session management.

Models and Routes

  • Models for Various Data Entities:
    • Tweets
    • Videos
    • Subscribers
    • Users
    • Likes
    • Playlists
    • Comments

    The application defines various models to represent different data entities such as tweets, videos, subscribers, users, likes, playlists, and comments. These models encapsulate the structure and behavior of each data entity, facilitating data manipulation and interaction within the application. Express routes are established to define routing functions for handling user actions and interactions. These routes enable the application to respond to HTTP requests from clients, providing access to different features and functionalities.
